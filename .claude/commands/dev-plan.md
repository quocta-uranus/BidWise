# Dev Plan

You are a senior frontend engineer and tech lead for this Next.js 15 + TypeScript monorepo. Your job is to create a structured implementation plan with live task tracking before ANY code is written.

## Usage

```
/dev-plan <description>
/dev-plan "Add folder upload retry with correct subfolder ID"
/dev-plan "Fix broken nav guard when switching Drive views"
```

- `<description>` — one sentence describing the feature or bug to work on
- If no description is given, ask the user for it before proceeding

---

## Step 1 — Classify the work

Determine if this is:
- **FEATURE** — new capability that doesn't exist yet
- **BUGFIX** — existing behavior that is broken
- **REFACTOR** — restructuring code without changing behavior
- **CHORE** — dependency update, config, tooling

> ⚠️ If this is a **BUGFIX**, remind the user: "Run `/dev-rca` first to document the root cause before starting this plan."

---

## Step 2 — Explore the codebase

Before writing the plan, read relevant files to understand:
- Which files will be touched (components, hooks, actions, constants, types)
- Existing patterns used in similar features (SWR, useCallback, useMemo, server actions)
- Integration points — what calls what
- Any existing tests that must stay green

Use `git diff develop...HEAD` or `git log --oneline -10` to understand current branch context.

---

## Step 3 — Write the plan document

Save to: `docs/tasks/{kebab-name}/PLAN.md`

Use this exact template:

```markdown
# Plan: {Title}

**Type:** FEATURE | BUGFIX | REFACTOR | CHORE
**Branch:** {current branch}
**Created:** {YYYY-MM-DD}
**Status:** 🟡 In Progress

> {One paragraph: what problem this solves and why it matters}

---

```

---

## Step 4 — Track tasks as you work

After creating the plan, **actively maintain PLAN.md** as you implement:

- Mark tasks `[x]` as each one is done
- Update **Status** in the header:
  - `🟡 In Progress` — work has started
  - `🔴 Blocked` — waiting on something, add a "Blocked by:" note
  - `✅ Done` — all tasks complete, acceptance criteria met

- After completing ALL tasks run:
  1. `/dev-review` — ensure no FAIL items
  2. `/dev-test` — generate or verify tests
  3. Update status to `✅ Done`

---

After creating the plan, print a summary:

```

```

If this is a BUGFIX also print:
```

```
